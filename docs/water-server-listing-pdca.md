# ウォーターサーバー広告 週次PDCA運用ルール

更新日: 2026-05-02

`/water-server/` のGoogle検索広告を、日額2,000円で検証するための運用ルールです。
目的は「広告をなんとなく触る」のを避け、毎週同じ項目・同じ基準で判断することです。

## 現在の前提

| 項目 | 内容 |
|---|---|
| LP | `https://vpscomparehub.com/water-server/` |
| 広告種別 | Google検索広告 |
| 初期目標 | ウェブサイトのトラフィック |
| 初期予算 | 2,000円/日 |
| 主な中間CV | 公式サイト/アフィリエイトリンクのCTAクリック |
| LP側イベント | `water_cta_click` |
| 改善周期 | 週次 |
| 初回チェック | 1日目に検索語句整理、7日目に本レビュー |

## 広告に出す掲載対象

広告を回すLPでは、アフィリエイトリンクが確認できている案件だけを表示する。

| サービス | 種別 | リンク状況 | メモ |
|---|---|---|---|
| オーケンウォーター | 宅配水/天然水/RO水 | A8リンク設定済み | 料金・契約条件比較との相性が良い |
| every frecious | 浄水型 | A8リンク設定済み | 定額制・ボトルなし訴求の主力 |
| フレシャス | 天然水 | A8リンク設定済み | 認知度とデザイン比較に使いやすい |
| ウォーターワン | 天然水 | AccessTradeリンク設定済み | 天然水・月額比較で使う |
| OCEAN | 水道直結型 | AccessTradeリンク設定済み | ボトルなし・使用量多め層向け |

未提携・未確認案件は、広告配信中のLPには戻さない。
全案件版に戻せる状態は、git tag `water-server-full-lineup-20260502` に保存済み。

## 毎週集めるデータ

| 取得元 | 必要な項目 |
|---|---|
| Google広告 | 費用、表示回数、クリック数、CTR、平均CPC、検索語句、キーワード別実績、広告アセット別実績 |
| GA4 / gtag | `water_cta_click` 数、CTA率、サービス別CTAクリック |
| A8 / AccessTrade | クリック、発生、確定、却下、報酬、案件別EPC/CVR |
| LP | 直近の変更点、リンク切れ、スマホ表示の違和感 |
| 市場/競合 | 価格改定、キャンペーン、競合LPの訴求変化 |

## 計測方針

ページビューだけをCVとして最適化しない。
LP訪問より、公式サイトへの送客クリックを重視する。

優先順位:

1. Google広告またはGA4で `water_cta_click` をCV化する。
2. `water_cta_click` を中間CVとして見る。
3. A8/AccessTradeの発生・確定を最終判断に使う。
4. `affiliate_key` で、どの案件に送客できているかを見る。

ページビューCPAが良くても、CTAクリックやASP成果がなければ改善対象。

## 初期キャンペーン設定

| 設定 | 方針 |
|---|---|
| ネットワーク | Week 1はGoogle検索のみ |
| 検索パートナー | OFF |
| ディスプレイネットワーク | OFF |
| 入札 | クリック数の最大化 |
| 上限CPC | 150〜250円から開始 |
| 日予算 | 2,000円 |
| 地域 | 日本 |
| 地域オプション | ターゲット地域にいるユーザー |
| 言語 | 日本語 |
| AI拡張/インテントマッチ | Week 1はOFF |
| 最終URL拡張 | OFF |
| 広告スケジュール | 初日は終日。データが出たら時間帯を確認 |

理由:

- 少額予算では、検索パートナーやディスプレイ混在の無駄クリックが痛い。
- クリック数の最大化は上限CPCで守る。
- 低予算では除外キーワードの精度がかなり重要。

## 広告文ルール

使いやすい表現:

- 料金と条件を比較
- 浄水型と宅配水を比較
- 一人暮らし向けも比較
- 解約条件まで確認
- 契約前の注意点を確認

確認なしで使わない表現:

- `公式`
- `最安`
- `No.1`
- `必ず`
- `絶対`
- 正確なキャンペーン金額/特典
- 未確認の赤ちゃん・医療・健康系表現
- 商標 + `怪しい` / `評判悪い` などのネガティブ訴求

## LPチェック条件

Google広告に出すLPは、単なるリンク集ではなく比較価値を出す。

最低条件:

- 比較表がある。
- 月額、契約期間、解約金、水の種類、ボトル交換の観点がある。
- PR/広告表記が見える。
- 運営者情報ページへ行ける。
- CTAリンクが正常に動く。
- スマホで読みやすい。
- 未確認の料金・キャンペーン表現がない。

## 週次PDCA

### Day 0: 配信開始

- LPがアフィリンクあり案件だけになっているか確認。
- ネットワーク、キーワード、除外、最終URLを確認。
- `water_cta_click` が発火するか確認。
- 2,000円/日で開始。

### Day 1: 無駄クリックだけ削る

見るもの:

- 明らかにズレた検索語句
- 既存ユーザー系、求人、修理、ログイン、説明書系
- 想定より高すぎるCPC
- 不承認やリンクエラー

Day 1では全体を大きく作り直さない。

### Day 3: 小さく調整

| 状況 | 対応 |
|---|---|
| 表示回数がほぼない | 上限CPCを50〜100円上げる、または狭いKWを1つ追加 |
| CTRが2%未満 | 検索意図に合わせて広告見出しを修正 |
| CPCが高い | 弱いキーワード停止、または上限CPCを下げる |
| 費用は出たがCTAゼロ | 検索語句とLPファーストビューを確認 |
| 特定案件だけCTAが強い | その案件の順位/広告文を強める |

### Day 7: 週次判断

`reports/water-server/weekly-pdca-template.md` を使って判断する。

| 状況 | 判断 |
|---|---|
| CTR高い、CTA低い | LPのファーストビュー/比較表が弱い |
| CTR低い、検索語句は良い | 広告文が弱い |
| 検索語句が悪い | 除外追加、マッチタイプを絞る |
| CTAはあるがASP成果なし | 案件順位・リンク先・訴求を確認 |
| ASP成果あり | 予算を20〜30%だけ増やす |
| 7,000〜10,000円使ってCTA傾向なし | 弱いKW停止、狭い意図で再テスト |

## 週次レポートの最後に必ず書くこと

1. 継続 / 停止 / 修正 / 予算増減の判断
2. 追加するキーワード
3. 停止するキーワード
4. 追加する除外キーワード
5. LPの修正点
6. 次の7日間の予算
