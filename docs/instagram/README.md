# Instagram 運用ディレクトリ（@lawyer.nakanohito）

このディレクトリは Instagram「弁護士の中の人（@lawyer.nakanohito）」の運用資料を保管します。
**台本作成・分析・新企画の起案は、すべてここを参照／更新して行う**こと。

---

## ディレクトリ構成

| パス | 用途 |
|---|---|
| `HANDOFF.md` | **必読**。アカウント概要・バズの法則・効かないパターン・過去ミス・運営者の判断スタイルなど全前提を集約 |
| `templates/` | 鉄板フォーマット3種のテンプレート（同じ◯◯でも変わるN選 / 裁判で出た言い訳 / 不倫×世界・歴史） |
| `scripts/` | 投稿した・する予定の台本。1ファイル1本。命名は `YYYYMMDD-slug.md` |
| `analytics/` | 数値分析・投稿リスト・被りチェック用のリスト |

---

## 新しい台本を作る前の必須手順

1. **`HANDOFF.md` を読む** — バズの法則、効かないパターン、運営者の好みを確認
2. **`analytics/posts-log.md` で被り・連続出演をチェック**
   - 同じテーマを3連続出していないか
   - 過去弾と同じケースを含んでいないか
3. **テーマの一貫性を確認** — 「弁護士」ワードを前面に出していないか、不倫から外れていないか
4. **テンプレートを `templates/` から複写** — 鉄板フォーマットに沿う
5. **Web検索で事実確認** — 判例・統計・法改正は必ず一次情報を当たる（生成だけはNG）
6. **画像生成プロンプトを末尾に必ず含める** — `HANDOFF.md` §6 のテンプレ

## 台本ファイルの基本構造

`scripts/` 配下に作るファイルは下記の構造で統一する。

```
# [投稿タイトル]

- フォーマット: [①ケース型 / ②言い訳ランキング型 / ③雑学型 / 議論型 / Q&A型 / 他]
- カテゴリ: [慰謝料/サイン・証拠/弁護士/探偵/統計・雑学/議論・コメント誘発/Q&Aナーチャリング]
- 作成日: YYYY-MM-DD
- 投稿日: (予定 or 実績)
- 関連弾: 第N弾（あれば）

## 1. フック（最初の0.5秒〜2秒）
- ◯◯字以内
- 引きの一文

## 2. ボディ（各ケース／各ランキング）
1. [見出し] — 説明
2. [見出し] — 説明
...

## 3. オチ／CTA
- まとめ
- 「@lawyer.nakanohito 詳しくはプロフのリンクへ」
  ※探偵系のみ：「@lawyer.nakanohito 信頼できる探偵はプロフのリンクへ」

## 4. キャプション本文
（投稿欄に貼る用、絵文字なし）

## 5. 画像生成プロンプト
（HANDOFF.md §6 のテンプレを冒頭に必ず入れる）

## 6. 参考ソース
- [URL/書籍/判例番号など]

## 7. 被りチェック結果
- 過去弾と被るケースなし／被りあり：◯◯（→修正済み）
```

## 分析の進め方

`analytics/posts-log.md` を **新規投稿のたびに更新**する。
- 投稿日 / タイトル / フォーマット / カテゴリ / 再生数 / いいね / コメント / 保存 / プロフ訪問 / リンクタップ
- 跳ねた要因・跳ねなかった要因の所感を1行メモ

定期分析（月1回想定）では、`analytics/insights.md` に「直近30日の傾向」「次に試すべき仮説」を書き溜める。

## 運営者の判断ルール（HANDOFF.md §12 抜粋・遵守）

- データ重視。感覚提案より過去数字との比較を出す
- 被り・矛盾のチェックは台本提出前に必ず実施
- 安易な「デザイン変更しよう」は却下対象。問題はコンテンツ側で解決
- 「ちゃんと調査して」が頻出。Web検索による一次情報確認は必須
